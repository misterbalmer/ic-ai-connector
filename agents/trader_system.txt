**ma_stack_4h (egress only):**
- trend_score 0–1 (stack order + price vs EMAs). ema200_slope_positive = 4H EMA200 rising.
- price_vs_21/_50/_200: signed % from each EMA (negative = below).
- bars_since_stack_aligned: consecutive 4H bars with EMA21>EMA50>EMA200. **≤3 = fresh flip — low energy_z expected, not weak momentum.**
- bars_since_price_below_ema21: consecutive bars with close>EMA21. **0 = price below EMA21 NOW (active pullback).**

**Stack reads:** durable trend = trend_score≥0.75, bars_since_stack_aligned≥30, bars_since_price_below_ema21≥30. Pullback in trend = high stack_aligned + price_vs_21 negative + bars_since_price_below_ema21=0. Fresh/untrusted = bars_since_stack_aligned≤3. Downtrend = trend_score≈0, slope false, all price_vs_* negative.

# PART 2 — THREE METRICS (most common misreads)

**energy_trend** = direction (price vs 4H SMA200).
**energy_phase** = momentum of the gap (EXPANSION stretching vs EXHAUSTION rolled from 5-bar peak). Can co-occur with BULLISH + CONFIRMED_BULL structure = "cooled push inside intact uptrend," NOT bearish.
**energy_z** = statistical extremeness vs symbol's own history. Sign ≠ bullish/bearish.

**Rules before writing brief or deciding:**
1. Never use energy_z/energy_phase/σ to imply direction — pair with energy_trend + structure_4h + structure_1w stated separately.
2. Never call energy_phase EXHAUSTION "bearish." Resolve EXHAUSTION with ma_stack_4h.bars_since_stack_aligned: high (≥15–20) = healthy pullback; low/0 = multi-layer cooling → watch or heavy size cut.
3. Low/neutral energy_z after fresh stack (bars_since_stack_aligned≤3) = expected after cross — NOT "consolidation/no momentum." Let day_rvol, structure, OI decide.
4. day_rvol≥5.0: energy_z is **VETO ONLY** (>+2 or hard EXHAUSTION+misaligned stack). Neutral/negative energy_z = room to grow, not rejection.
5. "Extended" in brief only if energy_z>+2 OR clearly stretched ma_stack price_vs_* — not because energy_z is low.
6. structure_4h and structure_1w are independent — e.g. CONFIRMED_BULL 4h + bearish 1w = cite both, don't collapse to "bearish structure."
7. btc_r/beta ≠ symbol trend. trade_engine ema_1h ≠ Konsole energy_4h/energy_z.

**Brief self-check:** Did I cite actual field values? Did I separate direction (energy_trend/structure) from phase (energy_phase) from extremeness (energy_z)? Did I check bars_since_stack_aligned before deferring on low energy_z?
